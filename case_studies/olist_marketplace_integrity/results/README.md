# Olist Results Guide

These files are generated by:

```bash
make olist-run
```

The Olist dataset is public marketplace data. It does not include confirmed fraud, abuse, sanctions, or enforcement labels. The outputs below are review-prioritization artifacts for marketplace integrity analysis.

## Files

### `01_dataset_profile.csv`

Table-level row counts used to confirm load coverage.

### `02_order_integrity_queue.csv`

Top 100 queued orders from `v_order_integrity_queue`. Each row includes order status, customer state, seller states, categories, value, freight, installments, review score, priority score, priority band, and risk flags.

Use this for: order-level review examples and triage workflow.

### `03_seller_integrity_rollup.csv`

Top 100 sellers by seller priority score. Includes order volume, delivered orders, canceled/unavailable orders, late deliveries, very late deliveries, low-review orders, average review score, late delivery rate, low-review rate, cancellation rate, and seller priority band.

Use this for: seller-quality review, marketplace operations, and partner-risk examples.

### `04_late_delivery_by_route.csv`

Customer-state to seller-state route analysis for single-seller delivered orders. Includes order count, late delivery count, average days late when late, late delivery rate, and average review score.

Use this for: fulfillment-route risk and customer-impact analysis.

### `05_low_review_patterns.csv`

Low-review concentration by customer state, category, and order status. Includes order count, low-review count, low-review rate, average days late, and average item value.

Use this for: customer experience review and category/status patterns.

### `06_payment_pattern_review.csv`

Payment-type and installment analysis. Includes order count, average payment value, average item value, average review score, canceled/unavailable count, and low-review count.

Use this for: payment-friction and installment-pattern review.

### `07_category_fulfillment_risk.csv`

Category-level fulfillment and review risk. Includes order count, late delivery count, canceled/unavailable count, low-review count, late delivery rate, low-review rate, and average item value.

Use this for: category-quality and fulfillment-risk prioritization.

### `08_case_study_metrics.csv`

Aggregate case-study metrics used in the summary, including total orders, queued order counts by band and flag, and seller counts by priority band.

Use this for: portfolio metrics and top-line case-study framing.

### `case_study_summary.md`

Markdown summary of the generated outputs with dataset profile, aggregate metrics, top queued orders, top seller rollups, route risk, low-review patterns, and interpretation notes.

Use this for: the portfolio site, project README, or case-study narrative.

## Responsible Interpretation

Queued records are not findings of fraud or abuse. They are examples of how SQL can turn public operational data into reviewable signals for analyst triage.
